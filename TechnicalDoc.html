Введение

JavaScript - это кроссплатформенный объектно-ориентированный язык сценариев. Это маленький и легкий язык. Внутри хост-среды (например, веб-браузера) JavaScript может быть связан с объектами его среды, чтобы обеспечить программный контроль над ними.
JavaScript содержит стандартную библиотеку объектов, таких как Array, Date и Math, и базовый набор языковых элементов, таких как операторы, управляющие структуры и операторы. Базовый JavaScript можно расширить для различных целей, дополнив его дополнительными объектами; например:

  • Клиентский JavaScript расширяет базовый язык, предоставляя объекты для управления браузером и его объектной моделью документов (DOM). Например, клиентские расширения позволяют приложению размещать элементы в HTML-форме и реагировать на пользовательские события, такие как щелчки мыши, ввод формы и навигация по страницам.
  • Серверный JavaScript расширяет базовый язык, предоставляя объекты, относящиеся к запуску JavaScript на сервере. Например, серверные расширения позволяют приложению взаимодействовать с базой данных, обеспечивать непрерывность информации от одного вызова к другому приложения или выполнять манипуляции с файлами на сервере.

Javascript и Java

JavaScript и Java в некотором роде похожи, но принципиально отличаются в некоторых других. Язык JavaScript напоминает Java, но не имеет статической типизации Java и строгой проверки типов. JavaScript придерживается большинства синтаксиса выражений Java, соглашений об именах и базовых конструкций потока управления, что и стало причиной его переименования из LiveScript в JavaScript.

В отличие от системы классов времени компиляции Java, созданной объявлениями, JavaScript поддерживает систему времени выполнения, основанную на небольшом количестве типов данных, представляющих числовые, логические и строковые значения. JavaScript имеет объектную модель на основе прототипов вместо более распространенной объектной модели на основе классов. Модель на основе прототипа обеспечивает динамическое наследование; то есть то, что наследуется, может варьироваться для отдельных объектов. JavaScript также поддерживает функции без каких-либо специальных декларативных требований. Функции могут быть свойствами объектов, выполняющимися как свободно типизированные методы.

JavaScript - это язык очень свободной формы по сравнению с Java. Вам не нужно объявлять все переменные, классы и методы. Вам не нужно беспокоиться о том, являются ли методы общедоступными, частными или защищенными, и вам не нужно реализовывать интерфейсы. Переменные, параметры и типы возвращаемых функций не являются явно типизированными.

Переменные

Вы используете переменные в качестве символических имен для значений в вашем приложении. Имена переменных, называемые идентификаторами, соответствуют определенным правилам.

Идентификатор JavaScript должен начинаться с буквы, подчеркивания (_) или знака доллара ($); последующие символы также могут быть цифрами (0-9). Поскольку JavaScript чувствителен к регистру, буквы включают символы от «A» до «Z» (заглавные буквы) и символы от «a» до «z» (строчные буквы).

В идентификаторах можно использовать буквы ISO 8859-1 или Unicode, такие как å и ü. Вы также можете использовать escape-последовательности Unicode в качестве символов в идентификаторах. Некоторые примеры допустимых имен: Number_hits, temp99 и _name.

Объявления переменных

Вы можете объявить переменную тремя способами:

С ключевым словом вар. Например,

var x = 42.

Этот синтаксис может использоваться для объявления как локальных, так и глобальных переменных.

Просто присвоив ему значение. Например,

x = 42.

Это всегда объявляет глобальную переменную. Он генерирует строгое предупреждение JavaScript. Вы не должны использовать этот вариант.

С ключевым словом пусть. Например,

let y = 13.

Этот синтаксис может использоваться для объявления локальной переменной области видимости блока. См. Области переменных ниже.

Области переменных

Когда вы объявляете переменную вне какой-либо функции, она называется глобальной переменной, потому что она доступна для любого другого кода в текущем документе. Когда вы объявляете переменную внутри функции, она называется локальной переменной, потому что она доступна только внутри этой функции.

JavaScript до ECMAScript 2015 не имеет области действия оператора блока; скорее, переменная, объявленная в блоке, является локальной для функции (или глобальной области видимости), в которой находится блок. Например, следующий код будет записывать 5, потому что область действия x - это функция (или глобальный контекст), в которой объявлен x, а не блок, который в этом случае является оператором if.

if (true) { var x = 5; } console.log(x); // 5

Это поведение изменяется при использовании объявления let, введенного в ECMAScript 2015.

if (true) { let y = 5; } console.log(y); // ReferenceError: y is not defined
Глобальные переменные

Глобальные переменные на самом деле являются свойствами глобального объекта. На веб-страницах глобальным объектом является окно, поэтому вы можете устанавливать глобальные переменные и получать к ним доступ, используя синтаксис window.variable.

Следовательно, вы можете получить доступ к глобальным переменным, объявленным в одном окне или фрейме из другого окна или фрейма, указав имя окна или фрейма. Например, если в документе объявлена переменная с именем phoneNumber, вы можете ссылаться на эту переменную из iframe как parent.phoneNumber.

Константы

Вы можете создать именованную константу только для чтения с ключевым словом const. Синтаксис идентификатора константы такой же, как и для идентификатора переменной: он должен начинаться с буквы, знака подчеркивания или доллара и может содержать буквы, цифры или символы подчеркивания.

const PI = 3.14;

Константа не может изменить значение посредством присваивания или быть повторно объявленной во время выполнения сценария. Он должен быть инициализирован значением.

Правила области видимости для констант такие же, как и для переменных области действия блока let. Если ключевое слово const опущено, предполагается, что идентификатор представляет переменную.

Вы не можете объявить константу с тем же именем, что и функция или переменная в той же области видимости. Например:

// THIS WILL CAUSE AN ERROR function f() {}; const f = 5; // THIS WILL
CAUSE AN ERROR ALSO function f() { const g = 5; var g; //statements}

Однако атрибуты объекта не защищены, поэтому следующий оператор выполняется без проблем.

const MY_OBJECT = {"key": "value"}; MY_OBJECT.key = "otherValue";
Типы данных

Последний стандарт ECMAScript определяет семь типов данных:

  • Шесть типов данных, которые являются примитивами:
    • Булиевы. true and false.
    • null. Специальное ключевое слово, обозначающее нулевое значение. Поскольку JavaScript чувствителен к регистру, null - это не то же самое, что Null, NULL или любой другой вариант.
    • undefined. Свойство верхнего уровня, значение которого не определено.
    • Число. 42 или 3.14159.
    • Строка. «Привет»
    • Символ (новый в ECMAScript 2015). Тип данных, экземпляры которого уникальны и неизменны.
  • и Объект

Хотя эти типы данных являются относительно небольшими, они позволяют вам выполнять полезные функции с вашими приложениями. Объекты и функции являются другими фундаментальными элементами языка. Вы можете рассматривать объекты как именованные контейнеры для значений, а функции как процедуры, которые может выполнять ваше приложение.

Оператор if...else

Используйте оператор if, чтобы выполнить оператор, если логическое условие истинно. Используйте необязательное предложение else для выполнения оператора, если условие ложно. Оператор if выглядит следующим образом:

if (condition) { statement_1; } else { statement_2; }

условием может быть любое выражение, которое оценивается как истинное или ложное. См. Boolean для объяснения того, что оценивается как true и false. Если условие оценивается как true, выполняется Statement_1; в противном случае, Statement_2 выполняется. Statement_1 и Statement_2 могут быть любым оператором, включая дополнительные вложенные операторы if.

Вы также можете составить операторы, используя else, если нужно последовательно протестировать несколько условий:

if (condition_1) { statement_1; } else if (condition_2) { statement_2;}
else if (condition_n) { statement_n; } else { statement_last; }

В случае нескольких условий, будет выполнено только первое логическое условие, которое оценивается как истинное. Чтобы выполнить несколько операторов, сгруппируйте их внутри оператора блока ({...}). Как правило, рекомендуется всегда использовать операторы блока, особенно при вложении операторов if:

if (condition) { statement_1_runs_if_condition_is_true;
statement_2_runs_if_condition_is_true; } else {
statement_3_runs_if_condition_is_false;
statement_4_runs_if_condition_is_false; }

Желательно не использовать простые назначения в условном выражении, потому что назначение может быть перепутано с равенством при просмотре кода. Например, не используйте следующий код:

if (x = y) { /* statements here */ }

Если вам нужно использовать присваивание в условном выражении, обычной практикой является установка вокруг скобок дополнительных скобок. Например:

if ((x = y)) { /* statements here */ }
Оператор while

Оператор while выполняет свои операторы до тех пор, пока указанное условие оценивается как true. Оператор while выглядит следующим образом:

while (condition) statement

Если условие становится ложным, оператор внутри цикла прекращает выполнение, и управление переходит к оператору, следующему за циклом.

Проверка условия происходит до выполнения оператора в цикле. Если условие возвращает true, оператор выполняется, и условие проверяется снова. Если условие возвращает false, выполнение останавливается, и управление передается в оператор, следующий за while.

Чтобы выполнить несколько операторов, используйте оператор блока ({...}) для группировки этих операторов.

Пример:

Следующий цикл while повторяется до тех пор, пока n меньше трех:

var n = 0; var x = 0; while (n < 3) { n++; x += n; }

С каждой итерацией цикл увеличивает n и добавляет это значение к x. Следовательно, x и n принимают следующие значения:

  • После первого прохода: n = 1 и x = 1
  • После второго прохода: n = 2 и x = 3
  • После третьего прохода: n = 3 и x = 6

После завершения третьего прохода условие n < 3 больше не выполняется, поэтому цикл завершается.

Объявление функций

Определение функции (также называемое объявлением функции или оператором функции) состоит из ключевого слова function, за которым следует:

  • Название функции.
  • Список аргументов функции, заключенный в скобки и разделенный запятыми.
  • Операторы JavaScript, которые определяют функцию, заключенную в фигурные скобки, {}.

Например, следующий код определяет простую функцию с именем square:

function square(number) { return number * number; }

Функция square принимает один аргумент, называемый числом. Функция состоит из одного оператора, который говорит, что должен возвращать аргумент функции (то есть число), умноженный на себя. Оператор return указывает значение, возвращаемое функцией.

return number * number;

Примитивные параметры (например, число) передаются в функции по значению; значение передается функции, но если функция изменяет значение параметра, это изменение не отражается глобально или в вызывающей функции.
